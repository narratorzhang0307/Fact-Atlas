# Fact Atlas · 知识星球

Fact Atlas 是一个可验证的个人知识地图与轻量混合式知识 DApp。FactRelay 是核验引擎，Signals 是每日发现全球知识的入口；三者共享同一份可追溯事实数据模型。

## 产品命题

人们会把读到的内容记进自己的知识体系，却很少保留它为什么可信、哪些证据反对它，以及结论何时可能失效。FactRelay 先用 Gonka Router 完成实时取证、双模型交叉核验和确定性评分；Fact Atlas 再把核验结果保存为私人事实节点。

知识有两条进入路径：

1. **Relay 主动探索：** 用户把书、电影、聊天、新闻或社交媒体中遇到的主张带进来。
2. **Signals 每日发现：** AI、科技、金融、气候能源、科学、健康、城市文化和公共政策等主题 Agent，每日扫描全球公开新闻并通过 Gonka 进行第一道重要性筛选。

两条路径都汇入 FactRelay 和 Evidence Council。经过深度核验后，用户可以把内容保存为设备内的私人节点，或明确发布到公共知识链。

一句话：**Build a knowledge world. Every fact keeps receipts. · 构建你的知识世界，让每条事实都保留回执。**

## 事实节点

每个私人或公共节点必须保留：

- 原始主张、结论、Truth Score 与核验时间；
- 来源账本、模型角色与 Gonka Request ID；
- 可选的地理坐标、地点标签与定位依据；
- `verified`、`disputed`、`uncertain` 或 `unplaced` 状态；
- 与其他节点之间可解释的空间、时间或主题关系。

公共节点额外保存 `ClaimKey`、精确快照哈希、证据根、回执根、评分策略哈希与记录哈希。公共节点按日期组成 Daily Edition，并可由用户钱包签名后把版本根发布到 `FactAtlasChronicle` 合约。完整模型见 [`KNOWLEDGE_CHAIN.md`](KNOWLEDGE_CHAIN.md)。

地理位置不是事实本身。不具备可靠空间语义的主张留在“未落位轨道”，不得为了视觉效果随机生成坐标。

## 交互与数据边界

产品使用三类稳定的交互与数据约束：

1. 移动端三 Tab、地球浏览与空间知识组织的交互模式；
2. 浏览器本地存储加发布订阅的私人节点模型，以及显式选择的公共 Chronicle 模型；
3. 地理、时间、标签和文字关系的可解释关联。

不引入照片、音乐、书影收藏或随机散点逻辑。Atlas 使用 Mapbox GL JS 和本项目的事实数据模型；地图只是组织层，不参与真实度计算。

## 地图呈现

- 中间 Atlas 使用 Mapbox `dark-v11` 深色地球底图；
- 节点按核验结论使用亮绿、亮粉、亮黄与亮蓝，节点内直接显示 Truth Score；
- 共享精确来源或 300 公里内的已确认地点才会连线；
- Mapbox 只获得可公开的 `pk.` 令牌，Gonka 密钥仍只在服务端；
- Mapbox 或网络暂不可用时，节点仍可保存，界面会诚实显示底图不可用。

## Gonka 边界

- 所有 AI 推理与信息验证继续只通过 Gonka Router；
- 公共检索、确定性评分、本地存储和地图投影不是 AI 推理；
- Gonka 回执证明调用来源，不等于事实为真；
- 用户决定写入私人星图或公共 Chronicle；完整私人内容与公开版本承诺保持分离。

## 完整知识闭环

1. 用户主动输入文本、链接或图片，或从主题 Agent 的每日情报中选择一条候选主张；
2. Gonka 通过 Kimi 调查与 MiniMax 质疑，输出可追溯核验结果；
3. Evidence Council 展示证据记录、调查方、质疑方与人工确认门；
4. 用户将结果保存为私人事实节点，或明确确认 canonical claim 并生成公共承诺；地点由用户明确确认；
5. 星图按结论显示节点，点击节点回到完整证据链；
6. 无可靠地点时仍可保存，但明确显示为未落位。

7. 公共节点按天形成 Merkle Edition，链接上一版本并保留修订历史；
8. 用户可导出 proof bundle，或用钱包签名把紧凑版本根写入 Chronicle 合约。

地图负责长期组织，知识链负责公开版本的完整性和演化顺序，FactRelay 负责证据核验。三者职责不互相替代。
